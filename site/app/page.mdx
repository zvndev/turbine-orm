# Turbine ORM

Postgres-native TypeScript ORM. Single-query nested relations via `json_agg`. One runtime dependency.

  <span style={{ color: 'var(--green)', userSelect: 'none' }}>$</span>

Turbine resolves an entire object graph in one database round-trip, no matter how deep the nesting. It ships with **Studio** — a read-only, loopback-bound web UI that your DBA will approve on sight (`BEGIN READ ONLY` + statement-stacking guard + 24-byte auth token, zero mutation surface). One runtime dependency (`pg`), ~110 KB on npm, and a one-import swap for edge runtimes. Deep `with` type inference lands in v1.0 — the runtime already nests relations for you today.

## A minimal example

```ts
import { turbine } from './generated/turbine';

const db = turbine({ connectionString: process.env.DATABASE_URL });

// Single query — users, their posts, and each post's comments
const users = await db.users.findMany({
  where: { orgId: 1 },
  with: {
    posts: {
      with: { comments: true },
      orderBy: { createdAt: 'desc' },
      limit: 5,
    },
  },
});

// users[0].posts[0].comments — fully typed, one round-trip
```

## Why Turbine

Prisma 7+ and Drizzle v2 also do single-query nested loads — the N+1 framing is dead. Turbine's value is architectural simplicity combined with measurable performance on the scenarios that matter.

- **One runtime dependency** (`pg`). No Rust binary, no WASM blob, no separate query engine process. ~110 KB on npm.
- **Read-only Studio.** The only TypeScript ORM Studio that physically cannot mutate your database — `BEGIN READ ONLY` + statement-stacking guard + loopback-only + 24-byte auth token. Your DBA will approve it on sight.
- **Real pipeline protocol.** `db.pipeline([...])` uses Postgres extended-query parse/bind/execute/sync in a single TCP flush — verified 2.58× speedup over sequential on Neon.
- **Edge-ready.** `turbine-orm/serverless` accepts any pg-compatible pool and runs on Vercel Edge, Cloudflare Workers, Deno Deploy.
- **Runtime-correct nested relations today.** Deep `with` clauses already return the full nested shape at runtime in a single round-trip. Type-level inference through `with` lands in v1.0; today you can use the generated `*With*` helper interfaces to annotate the return type when you need narrowing.
- **Honest benchmarks.** Tested against Prisma 7.6 and Drizzle 0.45 on a real pooled Neon database. Wins L2 nested reads by 1.59× over Drizzle, matches Prisma on streaming, beats Drizzle by 1.49×. See [Benchmarks](/benchmarks).
